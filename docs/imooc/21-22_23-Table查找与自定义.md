## 2. 知识点 + 演示复盘

### 2.1 21-22：查找

#### 前端：VeeValidate 搜索表单（UInput + UButton）

讲师特意**不用 `UForm`**：搜索框就一个 input + 按钮、要左右 inline，自定义布局更自由，而我们对 VeeValidate 已经很熟。craft-admin 同样：

```vue
<!-- users/index.vue 顶部加搜索条（template 节选）-->
<template>
  <div>
    <!-- 搜索条 -->
    <form class="mb-4 flex items-center gap-2" @submit="onSearch">
      <UInput
        v-model="keyword"
        v-bind="keywordAttrs"
        placeholder="按用户名搜索"
        icon="i-lucide-search"
        :color="errors.keyword ? 'error' : 'neutral'"
      />
      <UButton type="submit" label="查询" />
      <UButton
        v-if="showClear"
        label="清除"
        color="neutral"
        variant="outline"
        @click="onClear"
      />
    </form>

    <UTable :data="rows" :columns="columns" :loading="status === 'pending'" />
    <!-- 分页同 21-20 -->
  </div>
</template>
```

```ts
// users/index.vue <script setup> 增补
import { z } from 'zod'
import { toTypedSchema } from '@vee-validate/zod'
import { useForm } from 'vee-validate'

// 搜索表单：关键字不能为空
const searchSchema = z.object({ keyword: z.string().min(1) })
const { handleSubmit, defineField, errors, resetForm } = useForm({
  validationSchema: toTypedSchema(searchSchema),
})
const [keyword, keywordAttrs] = defineField('keyword')

// 关键：用单独的 searchText 触发请求，而不是直接用 keyword（v-model 会一直变）
const searchText = ref('')

const onSearch = handleSubmit((values) => {
  // 验证通过才更新 searchText（同时改两个 ref，useFetch 会合并成一次请求）
  searchText.value = values.keyword
  currentPage.value = 1 // 搜索回到第一页
})

const showClear = computed(() => searchText.value !== '')

const onClear = () => {
  searchText.value = ''
  currentPage.value = 1
  resetForm() // 清空表单 + 错误状态
}
```

把 `searchText` 加进 useFetch 的 query：

```ts
const { data, status } = await useFetch('/api/users', {
  query: {
    currentPage,
    pageSize,
    orderBy: computed(() => sort.value.column),
    order: computed(() => sort.value.direction),
    keyword: computed(() => searchText.value), // ← 新增
  },
  headers: useRequestHeaders(['cookie']),
})
```

> 💡 为什么不用 `keyword`（v-model）直接进 query？因为 v-model 每敲一个字都变，会狂发请求。用 `searchText` 只在点"查询"（校验通过）时更新 → 只发一次请求。
> 同时改 `searchText` 和 `currentPage` 两个 ref，useFetch 会**合并成一次**请求（不会发两次）。

> ⚠️ 依赖版本差异（UInput v2 → v4）
>
> | | 课程 v2 | craft-admin v4 |
> | --- | --- | --- |
> | 错误态颜色 | `color` 传 `gray`/`red` | `color` 语义化：`neutral`/`error`（v4 主题色名） |
> | 图标 | `icon` | `icon`（一致，`i-lucide-search`） |

#### 后端：MongoDB `$regex` 模糊匹配

```ts
// server/api/users/index.ts（21-22 版，新增 keyword 查找）
import type { SortOrder } from 'mongoose'

export default defineAuthResponseHandler(async (event) => {
  const query = getQuery(event)
  const currentPage = Number(query.currentPage ?? 1)
  const pageSize = Number(query.pageSize ?? 10)
  const orderBy = (query.orderBy as string) ?? 'createdAt'
  const order = (query.order as string) ?? 'desc'
  const keyword = (query.keyword as string) ?? ''

  const skip = (currentPage - 1) * pageSize
  const sort = { [orderBy]: order as SortOrder }

  // 有 keyword 就按 username 模糊搜，没有就查全部
  const findCondition = keyword
    ? { username: { $regex: keyword } }
    : {}

  const data = await UserSchema
    .find(findCondition)
    .skip(skip)
    .limit(pageSize)
    .sort(sort)
    .select('username nickName type role disabled createdAt updatedAt')
    .lean()
  const total = await UserSchema.countDocuments(findCondition) // total 也要带条件

  return { data, total, currentPage, pageSize }
})
```

> ⚠️ 易错：`total` 也要带 `findCondition`，否则搜索后分页总数还是全量，页数算错。

验证：输入 `viking` → 点查询 → 只显示用户名含 viking 的；点清除 → 恢复全部、表单状态重置。

### 2.2 21-23：自定义单元格（禁用开关 + PATCH 接口）

需求：加一列"禁用"，用开关控制用户 `disabled` 字段，点击发请求更新。

#### Step 1 model + 类型加 `disabled`

```ts
// shared/types/user.ts —— UserDataProps 加一行
export interface UserDataProps {
  // ...原字段
  disabled?: boolean
}
```

```ts
// server/models/user.ts —— schema 加 disabled
export const UserSchema = defineMongooseModel<UserDataProps>({
  name: 'User',
  schema: {
    // ...原字段
    disabled: { type: Boolean, default: false },
  },
  options: { /* ... */ },
})
```

> 💡 后端 `index.ts` 的 `.select(...)` 记得**带上 `disabled`**（上面 2.1 的 select 已含），否则列表里取不到。

#### Step 2 PATCH 接口

```ts
// packages/craft-admin/server/api/users/[id].patch.ts
export default defineAuthResponseHandler(async (event) => {
  const id = getRouterParam(event, 'id') // 路径参数 :id
  const { disabled } = await readBody<{ disabled: boolean }>(event)

  // findByIdAndUpdate + { new: true } 返回更新后的最新文档
  const newUser = await UserSchema.findByIdAndUpdate(id, { disabled }, { new: true })
  return newUser?.toJSON()
})
```

> ⚠️ Nuxt 4 差异（文件名约定）
> 动态路由 + 指定 HTTP 方法：文件名 `[id].patch.ts`（`[id]` 是路径参数，`.patch` 限定 PATCH 方法）。位置 `server/api/users/`。用 `defineAuthResponseHandler` 保护（修改操作需登录）。

#### Step 3 前端：`#<accessorKey>-cell` slot + USwitch

```vue
<!-- users/index.vue：UTable 加 disabled 列的自定义单元格 -->
<template>
  <UTable :data="rows" :columns="columns" :loading="status === 'pending'">
    <!-- v4 自定义单元格 slot：#<accessorKey>-cell -->
    <template #disabled-cell="{ row }">
      <USwitch
        :model-value="row.original.disabled"
        @update:model-value="(value) => toggleDisabled(value, row.original._id)"
      />
    </template>
  </UTable>
</template>
```

```ts
// columns 加一列
const columns: TableColumn<UserDataProps>[] = [
  // ...原列
  { accessorKey: 'disabled', header: '禁用' },
]

// 点击开关 → PATCH 更新 → 刷新列表
const toggleDisabled = async (disabled: boolean, id: string) => {
  await $fetch(`/api/users/${id}`, { method: 'PATCH', body: { disabled } })
  await refresh() // useFetch 返回的 refresh，重新拉当前页
}
```

> 注意：`useFetch` 解构出 `refresh` 一起用：`const { data, status, refresh } = await useFetch(...)`。
> `row.original` 才是这一行的原始数据（v4），`row.original._id` 拿到记录 id（`lean()` 默认含 `_id`）。

> ⚠️ 依赖版本差异（自定义单元格 + 开关组件 v2 → v4）
>
> | | 课程 v2 | craft-admin v4 |
> | --- | --- | --- |
> | 自定义单元格 slot | `#<key>-data="{ row }"` | **`#<accessorKey>-cell="{ row }"`** |
> | 行数据 | `row`（直接） | **`row.original`** |
> | 开关组件 | `UToggle` | **`USwitch`** |
> | 开关事件 | `@update:model-value` | `@update:model-value`（一致） |

验证：表格出现"禁用"列开关；点击 → Network 看到 `PATCH /api/users/:id`、`body: { disabled: true }`；刷新后状态持久化。

## 4. 易错点 + 关键 API 速查

### 4.1 易错点 Checklist

1. **`total` 要带 `findCondition`**：搜索后总数也要按条件算，否则页数错。
2. **搜索用 `searchText` 不用 `keyword`**：`keyword`（v-model）一直变会狂发请求；`searchText` 只在点查询时更新。
3. **同时改多个 ref 只发一次请求**：`searchText` + `currentPage` 一起改，useFetch 合并。
4. **自定义单元格 slot 是 `#<accessorKey>-cell`**（v4），不是 v2 的 `#<key>-data`；行数据用 **`row.original`**。
5. **`UToggle` 改名 `USwitch`**。
6. **`disabled` 字段要在 `.select()` 里带上**，否则列表取不到、开关恒为 false。
7. **PATCH 文件名 `[id].patch.ts`**，`getRouterParam(event, 'id')` 取参；`findByIdAndUpdate(..., { new: true })` 返回最新值。
8. **`refresh()` 从 useFetch 解构**：toggle 后刷新当前页；也可只改本地数据避免整页刷新。

### 4.2 关键 API 速查

| API | 作用 | 来源 |
| --- | --- | --- |
| `<UInput v-model :color icon>` | 搜索输入框 | NuxtUI v4 |
| `#<accessorKey>-cell="{ row }"` | 自定义单元格（`row.original`） | UTable slot |
| `<USwitch :model-value @update:model-value>` | 开关 | NuxtUI v4 |
| `{ username: { $regex: keyword } }` | MongoDB 模糊匹配 | mongoose |
| `getRouterParam(event, 'id')` | 取路径参数 | h3 |
| `readBody<T>(event)` | 读请求体 | h3 |
| `findByIdAndUpdate(id, data, { new: true })` | 更新并返回最新 | mongoose |
| `refresh()` | 重新执行 useFetch | Nuxt |
| `resetForm()` | 重置 VeeValidate 表单 | vee-validate |

### 4.3 与课程相比，本章**改写**了什么

- 自定义单元格：`#key-data` + `row` → `#accessorKey-cell` + `row.original`
- 开关：`UToggle` → `USwitch`
- 输入框错误色：`red` → `error`（v4 语义色）
- PATCH 接口用 `defineAuthResponseHandler` 保护
- `disabled` 类型放 `shared/types/`
