## 1. 概述

用户管理表格是后台最典型场景。讲师把表格功能拆成循序渐进的五步：**展示 → 分页 → 排序 → 查找 → 自定义操作**。本篇做前三步（查找/自定义见 21-22_23）。

核心思路（craft-admin 沿用课程）：**服务端驱动** —— 分页/排序/查找都把参数发给后端，后端用 MongoDB 处理后返回当前页数据；前端 `UTable` 只负责展示当前页（不用 TanStack 的客户端分页/排序）。

页面：`app/pages/users/index.vue`；接口：`server/api/users/index.ts`。

### 2.1 21-19：展示数据（UTable v4 + dayjs）

#### 后端：返回需要的字段

```ts
// packages/craft-admin/server/api/users/index.ts（21-19 版）
export default defineAuthResponseHandler(async () => {
  // 列表需要登录才能看 → 用 21-12 的高阶函数包裹
  const users = await UserSchema
    .find({})
    .select('username nickName type role createdAt updatedAt')

```

> ⚠️ 架构差异
> 课程列表接口用普通 `defineEventHandler`（靠全局中间件保护）。craft-admin 用 **`defineAuthResponseHandler`**（21-12 高阶函数）显式保护这个"需登录"接口。

#### 前端：UTable 展示

```vue
<!-- packages/craft-admin/app/pages/users/index.vue（21-19 版）-->
<template>
  <UTable :data="rows" :columns="columns" :loading="status === 'pending'" />
</template>

<script setup lang="ts">
import type { TableColumn } from '@nuxt/ui'
import dayjs from 'dayjs'

// 受保护接口：SSR 首屏需透传 cookie（同 21-14）
const { data, status } = await useFetch('/api/users', {
  headers: useRequestHeaders(['cookie']),
})

// dayjs 把 ISO 时间格式化成易读格式
const rows = computed(() =>
  (data.value ?? []).map(u => ({
    ...u,
    createdAt: dayjs(u.createdAt).format('YYYY-MM-DD HH:mm'),
    updatedAt: dayjs(u.updatedAt).format('YYYY-MM-DD HH:mm'),
  })),
)

// v4 列定义：TanStack 的 accessorKey + header（不再是 {key,label}）
const columns: TableColumn<UserDataProps>[] = [
  { accessorKey: 'username', header: '用户名' },
  { accessorKey: 'nickName', header: '昵称' },
  { accessorKey: 'type', header: '类型' },
  { accessorKey: 'role', header: '角色' },
  { accessorKey: 'createdAt', header: '创建时间' },
  { accessorKey: 'updatedAt', header: '更新时间' },
]
</script>
```

> ⚠️ 依赖版本差异（UTable v2 → v4，本章核心）
>
> | | 课程 NuxtUI v2 | craft-admin NuxtUI v4 |
> | --- | --- | --- |
> | 数据 prop | `:rows="data"` | **`:data="rows"`** |
> | 列定义 | `[{ key, label, sortable }]` | **`TableColumn[]`：`[{ accessorKey, header, cell }]`** |
> | 列类型 | 无 | `import type { TableColumn } from '@nuxt/ui'` |
> | loading | `:loading` | `:loading`（一致） |
> | 底层 | 自研 | TanStack Table |

> ⚠️ 依赖：dayjs
> `pnpm --filter @my-lego/craft-admin add dayjs`。课程也用 dayjs 格式化 MongoDB 的 ISO 时间。轻量（~2KB），无需类型包。

> ⚠️ Nuxt 4 差异（useFetch 状态）
> Nuxt 4 的 `useFetch` 返回 `status`（`'idle'|'pending'|'success'|'error'`），用 `status === 'pending'` 作 loading（旧的 `pending` 布尔仍在但推荐用 `status`）。

### 2.2 21-20：分页（后端 skip/limit + UPagination + 响应式 refetch）

#### 后端：skip/limit/total

```ts
// server/api/users/index.ts（21-20 版）
export default defineAuthResponseHandler(async (event) => {
  const query = getQuery(event)
  // query 里取出来都是 string，转成 number
  const currentPage = Number(query.currentPage ?? 1)
  const pageSize = Number(query.pageSize ?? 10)

  const skip = (currentPage - 1) * pageSize // 跳过前面页

  const data = await UserSchema
    .find({})
    .skip(skip)
    .limit(pageSize)
    .select('username nickName type role createdAt updatedAt')
    .lean()

  const total = await UserSchema.countDocuments({}) // 总条数

  return { data, total, currentPage, pageSize }
})
```

> 💡 返回结构从"数组"改成 `{ data, total, currentPage, pageSize }`，是通用分页列表的标准形态。

#### 前端：UPagination + useFetch 响应式 query 自动 refetch

```vue
<!-- users/index.vue（21-20 版）-->
<template>
  <div>
    <UTable :data="rows" :columns="columns" :loading="status === 'pending'" />

    <div class="mt-4 flex items-center justify-between">
      <UBadge color="neutral" variant="subtle">共 {{ data?.total ?? 0 }} 条</UBadge>
      <UPagination
        v-model:page="currentPage"
        :items-per-page="pageSize"
        :total="data?.total ?? 0"
      />
    </div>
  </div>
</template>

<script setup lang="ts">
import type { TableColumn } from '@nuxt/ui'
import dayjs from 'dayjs'

const currentPage = ref(1)
const pageSize = ref(10)

// query 传响应式：currentPage 变化时 useFetch 自动重新请求（替代手写 watch + refresh）
const { data, status } = await useFetch('/api/users', {
  query: { currentPage, pageSize },
  headers: useRequestHeaders(['cookie']),
})

const rows = computed(() =>
  (data.value?.data ?? []).map(u => ({
    ...u,
    createdAt: dayjs(u.createdAt).format('YYYY-MM-DD HH:mm'),
    updatedAt: dayjs(u.updatedAt).format('YYYY-MM-DD HH:mm'),
  })),
)
// columns 同上
</script>
```

> ⚠️ 依赖版本差异（UPagination v2 → v4）
>
> | | 课程 v2 | craft-admin v4 |
> | --- | --- | --- |
> | 当前页 | `v-model` | **`v-model:page`** |
> | 每页条数 | `:page-count`（=每页多少条） | **`:items-per-page`** |
> | 总数 | `:total` | `:total`（条目数，组件自己算页数） |

> 💡 **useFetch 的"响应式 query 自动 refetch"**：把 `currentPage`/`pageSize` 这样的 ref 放进 `query`，它们变化时 useFetch 会自动重新请求 —— 这正是课程里讲的 "computed URL / watch" 的等价能力（课程用 computed URL，craft-admin 用 `query` 传 ref，更直观）。

### 2.3 21-21：排序（后端 sort + 前端自定义 header 排序按钮）

排序也是**服务端驱动**：点击列头 → 改排序参数 → useFetch 重新请求后端排序后的数据（不是前端对当前 10 条排序）。

#### 后端：MongoDB sort

```ts
// server/api/users/index.ts（21-21 版，新增排序）
import type { SortOrder } from 'mongoose'

export default defineAuthResponseHandler(async (event) => {
  const query = getQuery(event)
  const currentPage = Number(query.currentPage ?? 1)
  const pageSize = Number(query.pageSize ?? 10)
  const orderBy = (query.orderBy as string) ?? 'createdAt'
  const order = (query.order as string) ?? 'desc'

  const skip = (currentPage - 1) * pageSize
  // MongoDB sort：{ 字段: 'asc' | 'desc' }
  const sort = { [orderBy]: order as SortOrder }

  const data = await UserSchema
    .find({})
    .skip(skip)
    .limit(pageSize)
    .sort(sort)
    .select('username nickName type role createdAt updatedAt')
    .lean()
  const total = await UserSchema.countDocuments({})

  return { data, total, currentPage, pageSize }
})
```

> 💡 `SortOrder` 是 mongoose 导出的类型（`'asc' | 'desc' | 1 | -1`），`order as SortOrder` 做类型断言。

#### 前端：自定义 header 渲染排序按钮

NuxtUI v4 的 UTable 列 `header` 可以是字符串，也可以是**渲染函数**（用 `h()` 返回组件）。我们在创建时间/更新时间两列放排序按钮，点击改 `sort` 响应式对象：

```vue
<script setup lang="ts">
import type { TableColumn } from '@nuxt/ui'
import dayjs from 'dayjs'

const currentPage = ref(1)
const pageSize = ref(10)
const sort = ref<{ column: string, direction: 'asc' | 'desc' }>({
  column: 'createdAt',
  direction: 'desc', // 默认按创建时间倒序（最新在前）
})

const { data, status } = await useFetch('/api/users', {
  query: {
    currentPage,
    pageSize,
    // computed：sort 变化时 query 变化 → 自动 refetch
    orderBy: computed(() => sort.value.column),
    order: computed(() => sort.value.direction),
  },
  headers: useRequestHeaders(['cookie']),
})

const rows = computed(() =>
  (data.value?.data ?? []).map(u => ({
    ...u,
    createdAt: dayjs(u.createdAt).format('YYYY-MM-DD HH:mm'),
    updatedAt: dayjs(u.updatedAt).format('YYYY-MM-DD HH:mm'),
  })),
)

// h() 里用组件要 resolveComponent
const UButton = resolveComponent('UButton')

const toggleSort = (column: string) => {
  if (sort.value.column === column) {
    sort.value.direction = sort.value.direction === 'asc' ? 'desc' : 'asc'
  }
  else {
    sort.value = { column, direction: 'asc' }
  }
  currentPage.value = 1 // 排序后回到第一页
}

// 生成"可排序列头"的渲染函数
const sortableHeader = (column: string, label: string) => () => {
  const active = sort.value.column === column
  return h(UButton, {
    color: 'neutral',
    variant: 'ghost',
    label,
    icon: active
      ? (sort.value.direction === 'asc' ? 'i-lucide-arrow-up' : 'i-lucide-arrow-down')
      : 'i-lucide-arrow-up-down',
    class: '-mx-2.5',
    onClick: () => toggleSort(column),
  })
}

const columns: TableColumn<UserDataProps>[] = [
  { accessorKey: 'username', header: '用户名' },
  { accessorKey: 'nickName', header: '昵称' },
  { accessorKey: 'type', header: '类型' },
  { accessorKey: 'role', header: '角色' },
  { accessorKey: 'createdAt', header: sortableHeader('createdAt', '创建时间') },
  { accessorKey: 'updatedAt', header: sortableHeader('updatedAt', '更新时间') },
]
</script>
```

> ⚠️ 依赖版本差异（排序范式 v2 → v4）
>
> | | 课程 v2 | craft-admin v4（服务端驱动） |
> | --- | --- | --- |
> | 开启排序 | 列 `sortable: true` | 列 `header` 用渲染函数放排序按钮 |
> | 排序状态 | `v-model:sort`（`{ column, direction }`） | 自己的 `sort` 响应式对象 |
> | 服务端模式 | `sort-mode="manual"` | 不接 TanStack 排序，直接改 `sort` → useFetch query refetch |
> | 默认排序 | `sort` 初值 | `sort` ref 初值 `{ createdAt, desc }` |
>
> 也可以用 v4 的 `v-model:sorting` + `:sorting-options="{ manualSorting: true }"` + `watch` 来做服务端排序（更"TanStack 原生"）。craft-admin 选**自定义 header 按钮**，因为更贴合课程"点击即请求后端"的直觉、代码也直白。

验证：默认按创建时间倒序；点击"创建时间"列头 → 发新请求（`orderBy=createdAt&order=asc`）→ 表格刷新；"更新时间"同理。

## 4. 易错点 + 关键 API 速查

### 4.1 易错点 Checklist

1. **`:data` 不是 `:rows`**；列是 `accessorKey`/`header` 不是 `key`/`label`。照 v2 写表格空白。
2. **`h()` 里用组件要 `resolveComponent('UButton')`**，不能直接写 `UButton`（在 `<script>` 里它不是值）。
3. **受保护接口 SSR 要透传 cookie**：`useFetch('/api/users', { headers: useRequestHeaders(['cookie']) })`，否则首屏 401（httpOnly，见 21-14）。
4. **服务端排序别用 TanStack 客户端排序**：当前页只有 10 条，客户端排序只排这 10 条。要把排序参数发后端。
5. **排序后回第一页**：`currentPage.value = 1`，否则可能停在不存在的页。
6. **`query` 里放 ref/computed 才会自动 refetch**；放普通值不会响应变化。
7. **`SortOrder` 从 `mongoose` import type**，`order as SortOrder` 断言。
8. **dayjs 格式化放 computed**，不要改 `data` 本身（保持数据源纯净）。

### 4.2 关键 API 速查

| API | 作用 | 来源 |
| --- | --- | --- |
| `<UTable :data :columns :loading>` | 表格展示 | NuxtUI v4 |
| `TableColumn<T>` | 列类型（accessorKey/header/cell） | `@nuxt/ui` |
| `header: () => h(UButton, {...})` | 自定义列头（排序按钮） | vue `h` + `resolveComponent` |
| `<UPagination v-model:page :items-per-page :total>` | 分页 | NuxtUI v4 |
| `<UBadge color variant>` | 展示 total | NuxtUI v4 |
| `useFetch(url, { query, headers })` | 响应式 query 自动 refetch + SSR cookie 透传 | Nuxt |
| `UserSchema.find().skip().limit().sort().lean()` | 分页 + 排序查询 | mongoose |
| `countDocuments(filter)` | 总条数 | mongoose |
| `SortOrder` | 排序方向类型 | `mongoose` |
| `dayjs(iso).format('YYYY-MM-DD HH:mm')` | 时间格式化 | dayjs |

### 4.3 与课程相比，本章**改写**了什么

- UTable：`:rows`/`{key,label}`/`v-model:sort`/`sortable` → `:data`/`TableColumn[{accessorKey,header}]`/自定义 header 排序
- UPagination：`v-model`/`:page-count` → `v-model:page`/`:items-per-page`
- 重新请求：computed URL → useFetch `query` 传 ref 自动 refetch
- 列表接口用 `defineAuthResponseHandler` 保护 + SSR cookie 透传
- 类型 `UserDataProps` 来自 `shared/types/`
