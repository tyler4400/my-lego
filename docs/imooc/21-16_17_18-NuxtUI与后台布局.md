## 2. 知识点 + 演示复盘

### 2.1 NuxtUI 是什么（v2 vs v4）

NuxtUI 是 Nuxt 官方组件库，**基于 Tailwind 做样式 + 一个无样式可访问性库做交互**：

| | 课程 NuxtUI v2 | craft-admin NuxtUI v4 |
| --- | --- | --- |
| 底层无样式库 | **Headless UI** | **Reka UI**（Radix Vue 后继） |
| Tailwind | v3 | v4 |
| 表格底层 | 自研 | **TanStack Table** |
| 组件前缀 | `U`（可配） | `U`（可配 `ui.prefix`） |
| 主题系统 | `app.config.ts` 的 `ui` | `@theme` + `app.config.ts`，`color` 语义化（primary/neutral...） |

理念一致：原子化 Tailwind + 无样式交互组件 + 高度可定制主题。

### 2.2 安装 NuxtUI v4（含从 21-1 的 Tailwind 配置迁移）

> ⚠️ 依赖版本差异（安装方式全变，且要改 21-1 的配置）
> 课程 v2：`npm install @nuxt/ui`，加进 `modules` 即可（它自动装 Tailwind v3）。
> craft-admin v4：NuxtUI **自己接管 Tailwind v4 集成**，所以要把 21-1 手动配的 `@tailwindcss/vite` 插件**撤掉**，改用 NuxtUI 的方式。

**Step 1 安装**

```bash
pnpm --filter @my-lego/craft-admin add @nuxt/ui
# tailwindcss 21-1 已装；@nuxt/ui 依赖它
```

**Step 2 改 `nuxt.config.ts`**：把 `@nuxt/ui` 加进 modules，**移除手动的 `vite.plugins: [tailwindcss()]`**（NuxtUI 接管）：

```ts
// packages/craft-admin/nuxt.config.ts（节选）
export default defineNuxtConfig({
  modules: [
    '@nuxt/ui', // ← 新增；它会自动处理 Tailwind v4 的 Vite 集成
    ['@vee-validate/nuxt', { /* ... */ }],
    ['nuxt-mongoose', { /* ... */ }],
  ],

  css: ['~/assets/css/main.css'],

  // ❌ 移除 21-1 的这段（NuxtUI 已接管 Tailwind）
  // vite: { plugins: [tailwindcss()] },
})
```

> 💡 `@tailwindcss/vite` 这个包可以从依赖里删掉（不再手动用）；留着也不报错。`tailwindcss` 包要保留（NuxtUI 依赖）。

**Step 3 改 `app/assets/css/main.css`**：加 `@import "@nuxt/ui";`

```css
/* packages/craft-admin/app/assets/css/main.css */
@import "tailwindcss";
@import "@nuxt/ui";
```

**Step 4 用 `<UApp>` 包裹 `app.vue`**：

```vue
<!-- packages/craft-admin/app/app.vue -->
<template>
  <UApp>
    <NuxtRouteAnnouncer />
    <NuxtLoadingIndicator />
    <NuxtLayout>
      <NuxtPage />
    </NuxtLayout>
  </UApp>
</template>
```

> ⚠️ Nuxt 4 / 版本差异
> `<UApp>` 是 NuxtUI v4 **新增的必需根组件**（提供 Toast / Tooltip / 程序化 Overlay 的全局上下文）。课程 v2 没有这个。包在最外层即可。
> NuxtUI v4 还自动注册了 `@nuxt/icon`、`@nuxt/fonts`、`@nuxtjs/color-mode`，**不用**手动加进 modules。

> ⚠️ 课程提到的"两个 Tailwind 版本冲突需删 node_modules 重装" —— 那是 v2 时代（@nuxt/ui 自带 tailwind 与你手动装的冲突）。craft-admin v4 这边的对应坑是 **`@tailwindcss/vite` 与 NuxtUI 的 Tailwind 集成并存**，按上面 Step 2 移除手动插件即可，不会冲突。

**Step 5 试用**：随便放个 `<UButton>` 验证：

```vue
<UButton label="Hello NuxtUI" icon="i-lucide-rocket" />
```

> ⚠️ 图标方案差异
> 课程 v2 用 [Iconify](https://icon-sets.iconify.design/) 的 `i-<collection>-<name>`（演示用 `i-heroicons-pencil-square`）。v4 同样走 Iconify，但官方示例多用 **`i-lucide-*`**（Lucide 图标集）。两者都能用，写法 `i-集合名-图标名`。

### 2.3 21-17：用 NuxtUI 做 Header

目标（讲师的 Mockup）：Header 左侧是 Logo/链接，右侧是**用户头像 + 点击展开的下拉菜单**（用户名 / 编辑资料 / 退出登录，带分割线）。

craft-admin 把它抽成 `app/components/GlobalHeader.vue`（21-3 的临时 Header 升级为正式组件）：

```vue
<!-- packages/craft-admin/app/components/GlobalHeader.vue -->
<template>
  <header class="border-b border-gray-200 bg-white">
    <div class="flex items-center justify-between px-6 py-3">
      <!-- 左：Logo / 链接 -->
      <NuxtLink to="/" class="text-lg font-semibold text-gray-800">
        craft-admin
      </NuxtLink>

      <!-- 右：头像 + 下拉菜单 -->
      <UDropdownMenu :items="items" :ui="{ content: 'w-52' }">
        <UAvatar
          :src="currentUser.data?.picture"
          :alt="currentUser.data?.username"
          icon="i-lucide-user"
          size="md"
          class="cursor-pointer"
        />
      </UDropdownMenu>
    </div>
  </header>
</template>

<script setup lang="ts">
import type { DropdownMenuItem } from '@nuxt/ui'

const currentUser = useCurrentUser()

const handleLogout = async () => {
  await $fetch('/api/users/logout', { method: 'POST' })
  currentUser.value.isLogin = false
  currentUser.value.data = null
  navigateTo('/login')
}

// 二维数组 = 分组（组间自动加分割线）
const items = computed<DropdownMenuItem[][]>(() => [
  [{ label: currentUser.value.data?.username ?? '未登录', avatar: { src: currentUser.value.data?.picture, icon: 'i-lucide-user' } }],
  [
    { label: '编辑个人资料', icon: 'i-lucide-pencil', to: '/profile' },
    { label: '退出登录', icon: 'i-lucide-log-out', onSelect: handleLogout },
  ],
])
</script>
```

> ⚠️ 依赖版本差异（下拉菜单组件改名 + API 改）
>
> | | 课程 v2 | craft-admin v4 |
> | --- | --- | --- |
> | 组件名 | `UDropdown` | **`UDropdownMenu`** |
> | 数据 prop | `:items`（二维数组） | `:items`（二维数组，结构类似） |
> | 点击回调 | item 的 **`click`** | item 的 **`onSelect`** |
> | 快捷键 | `shortcut` | `kbds` |
> | 弹出位置 | `:popper="{ placement }"` | `:content="{ side, align }"` 或默认 |
> | 触发器 | 默认 slot | 默认 slot（这里放 `UAvatar`） |

> ⚠️ Monorepo / 架构整合
> craft-admin 的"退出登录"接上了真实逻辑：调 21-12 的 `logout` 接口（清 httpOnly cookie）+ 清空 `useCurrentUser` 内存态 + 跳登录页。课程这节只渲染界面、不接功能。

> 💡 `UAvatar` 的 `icon` 作为 `src` 为空时的 fallback（用户没头像就显示用户图标）。

### 2.4 21-18：剩余布局（Sidebar + 内容区）

#### 先修正宽度

课程发现外层 `container` 限制了宽度（1280px），后台要占满整屏。craft-admin 的 `default.vue` 直接用全宽布局。

#### Sidebar 组件（NuxtUI 垂直导航）

```vue
<!-- packages/craft-admin/app/components/Sidebar.vue -->
<template>
  <UNavigationMenu
    orientation="vertical"
    :items="items"
    class="w-full"
  />
</template>

<script setup lang="ts">
import type { NavigationMenuItem } from '@nuxt/ui'

const items: NavigationMenuItem[] = [
  { label: '用户管理', icon: 'i-lucide-users', to: '/users' },
  { label: '海报管理', icon: 'i-lucide-image', to: '/posters' },
  { label: '模板管理', icon: 'i-lucide-layout-template', to: '/templates' },
]
</script>
```

> ⚠️ 依赖版本差异（垂直导航组件改名）
>
> | | 课程 v2 | craft-admin v4 |
> | --- | --- | --- |
> | 组件名 | `UVerticalNavigation` | **`UNavigationMenu`** + `orientation="vertical"` |
> | 数据 prop | `:links`（`[{ label, icon, to }]`） | `:items`（`[{ label, icon, to }]`，字段类似） |

#### 组装 `default.vue`（Header + 左右两栏）

```vue
<!-- packages/craft-admin/app/layouts/default.vue -->
<template>
  <div class="min-h-screen flex flex-col">
    <GlobalHeader />

    <!-- 左右两栏：左侧定宽 Sidebar，右侧 flex-1 内容区 -->
    <div class="flex flex-1">
      <aside class="w-[250px] border-r border-gray-200 px-2 pt-3">
        <Sidebar />
      </aside>
      <main class="flex-1 bg-gray-50 px-4 pt-4">
        <slot />
      </main>
    </div>
  </div>
</template>
```

> 💡 课程在这里又讲了 Tailwind 的**任意值**：侧边栏要精确 300px（预设 `w-72`=288px 不够），用 `w-[300px]`（craft-admin 这里用 `w-[250px]`）。这与 21-2 的 arbitrary values 一致。

> ⚠️ Nuxt 4 差异
> `GlobalHeader`、`Sidebar` 在 `app/components/` 下，Nuxt 自动注册为全局组件，`default.vue` 里直接用、无需 import。课程在项目根 `components/`。

验证：访问任意业务页，看到顶部 Header（头像下拉可展开）+ 左侧导航 + 右侧内容；点击"用户管理"跳 `/users`。

## 3. 三维差异速查表

### 3.1 Nuxt 3 → Nuxt 4 差异

| 项目              | 课程（Nuxt 3）          | craft-admin（Nuxt 4）          |
| ----------------- | ----------------------- | ------------------------------ |
| 组件位置          | `components/`            | `app/components/`              |
| 布局位置          | `layouts/default.vue`    | `app/layouts/default.vue`       |
| `app.vue`         | 项目根                  | `app/app.vue`（包 `<UApp>`）    |

### 3.2 依赖版本差异（NuxtUI v2 → v4，本章核心）

| 维度              | 课程 NuxtUI v2            | craft-admin NuxtUI v4                       |
| ----------------- | ------------------------ | ------------------------------------------- |
| 安装              | `@nuxt/ui`（自动装 Tailwind v3） | `@nuxt/ui` + 接管 Tailwind v4（移除手动 vite 插件） |
| CSS 入口          | 无需手动 import           | `@import "tailwindcss"; @import "@nuxt/ui";` |
| 根组件            | 无                        | **`<UApp>` 必需**                            |
| 底层交互库        | Headless UI               | Reka UI                                     |
| 下拉菜单          | `UDropdown`（item.click） | `UDropdownMenu`（item.onSelect）             |
| 垂直导航          | `UVerticalNavigation`（:links） | `UNavigationMenu orientation="vertical"`（:items） |
| 头像              | `UAvatar`                 | `UAvatar`（`icon` 作 fallback）              |
| 图标              | `i-heroicons-*`           | `i-lucide-*`（仍是 Iconify，集合名任选）      |

### 3.3 Monorepo / 架构差异

| 维度       | 课程            | craft-admin                                          |
| ---------- | --------------- | ---------------------------------------------------- |
| 退出登录   | 只渲染、不接功能 | 调 `logout` 接口 + 清 `useCurrentUser` + 跳登录       |
| Header 用户信息 | 占位          | 取 `useCurrentUser().data`（21-13 的登录态）          |

## 4. 易错点 + 关键 API 速查

### 4.1 易错点 Checklist

1. **装 NuxtUI v4 后移除手动 `@tailwindcss/vite` 插件**，否则可能与 NuxtUI 的 Tailwind 集成冲突。
2. **`main.css` 要 `@import "@nuxt/ui";`**，漏了组件没样式。
3. **`<UApp>` 必须包根**，否则 Toast/Tooltip/Overlay 不工作。
4. **组件名都变了**：`UDropdown`→`UDropdownMenu`、`UVerticalNavigation`→`UNavigationMenu`。照课程 v2 写会"组件不存在"。
5. **下拉项回调 `click`→`onSelect`**，写 `click` 不触发。
6. **图标要装图标集**：`i-lucide-*` 需要 Iconify 的 lucide 集合（NuxtUI v4 一般已含；缺图标时按提示装 `@iconify-json/lucide`）。
7. **`app/components/` 新建后重启 dev server**（GlobalHeader/Sidebar 首次创建）。

### 4.2 关键组件速查（v4）

| 组件 | 关键 props / 用法 |
| --- | --- |
| `UApp` | 根组件，提供全局上下文 |
| `UButton` | `label` / `icon` / `color` / `variant` / `@click` |
| `UDropdownMenu` | `:items`（二维分组）/ 默认 slot 作触发器；item: `{ label, icon, avatar, kbds, onSelect, to, children }` |
| `UNavigationMenu` | `:items` / `orientation="vertical"`；item: `{ label, icon, to }` |
| `UAvatar` | `:src` / `:alt` / `icon`(fallback) / `size` |
| 图标 | `i-<collection>-<name>`，如 `i-lucide-user` |

### 4.3 与课程相比，本章**改写**了什么

- NuxtUI v2 → v4：安装、`<UApp>`、CSS import、组件名（`UDropdownMenu`/`UNavigationMenu`）、item 回调（`onSelect`）
- 移除 21-1 的手动 `@tailwindcss/vite`，改由 NuxtUI 接管 Tailwind v4
- 退出登录接上真实 `logout` 接口 + `useCurrentUser`
- 组件/布局路径在 `app/` 下
